Current Status

Still under major development. The high level goal is to make cross platform C++ development not suck, by making it as easy to use 3rd party software and describe dependencies with C++ as it is for Python or Node.js developers.

Roadmap

Really rough roadmap

  • Multi-distro linux support
  • Full distro independent set of development packages
  • chroot package building
  • support for X11 and OpenGL apps

Then things get a little more fuzzy on the exact ordering but I am considering all of these features:

  • Explicit cross platform support
  • One description file producing multiple packages
  • Support for BSD
  • Support for Mac OS X
  • Support for Windows 7/8
